# Log Anomalies & System Analysis Report

This document summarizes identified problems, their causes, and applied or proposed fixes.

## 1. Duplicate Notifications & Processing
**Problem**: Users received the same listing multiple times in a single cycle. Listings were also being saved twice.
**Root Cause**: The Facebook scraper scrolls and collects posts; dynamic loading sometimes causes the same post to appear twice in the DOM. Additionally, different scraper groups might contain overlapping posts.
**Fix (Applied)**:
- Added set-based deduplication in `main.py` immediately after collection.
- Unique-ified skipped IDs in `SeenListingsRepository` logs.

## 2. Price Parsing Confusions
**Problem**: Phone numbers (e.g., `0506663600`) were incorrectly extracted as prices.
**Root Cause**: The AI engine's prompt was not specific enough about excluding 10-digit number patterns from the price field.
**Fix (Applied)**:
- Hardened the AI prompt in `core/ai_engine.py` with explicit "Negative constraints" for 10-digit numbers and examples of what to ignore.

## 3. Yad2 Bot Detection (Complicated)
**Problem**: Yad2 redirects to `validate.perfdrive.com`, causing extraction to fail with "No __NEXT_DATA__ found".
**Analysis**: This is a sophisticated anti-bot measure. Simple header adjustments may not be sufficient for long-term reliability.
**Fix (Attempted)**:
- Updated headers to include modernized browser parameters (`Sec-Fetch`, `Referer`, etc.).
**Recommendation**: If headers fail, switching to a Playwright-based scraper (like the Facebook one) or using a residental proxy might be necessary.

## 4. Missing Facebook URLs
**Problem**: Some listings had `URL=[MISSING]`.
**Root Cause**: Permalinks in Facebook's desktop DOM use various formats (e.g., `fbid=`, `/groups/.../permalink/`) which were not all captured by the initial selectors.
**Fix (Applied)**:
- Expanded the URL collection logic in `FacebookScraper` to handle multiple href patterns and relative paths.
